## 2. Round 1固定修复范围

### B01：Persistence Workflow拥有事务

正式入口不得要求调用方手工commit。

目标：

```text
Workflow接收sessionmaker或DB factory
→ 内部创建Session
→ 内部commit/rollback
```

业务对象事务：

```text
Entity + DataPoint + Claim + Evidence
→ 单一原子事务
```

### B02：ProcessingRun独立事务

必须实现：

```text
事务1：ProcessingRun RUNNING并commit
事务2：业务对象写入
事务3：SUCCEEDED或FAILED并commit
```

真实故障注入：

```text
写入第N个对象时抛异常
→ 业务对象数量=0
→ FAILED ProcessingRun仍存在
→ 新Session可查询
```

正式Workflow必须调用该路径，不能只保留未使用辅助函数。

### B03：ReviewBundle完整Preflight

持久化前：

```text
重算bundle_sha256并匹配
校验ContextWindow Hash
校验accepted/rejected集合
校验目标Candidate无ERROR Finding
校验source_unit_id存在
校验Document一致
校验Workspace一致
校验Candidate引用可解析
校验Provider/Profile允许列表
```

任一失败：

```text
数据库业务写入=0
```

### B04：SourceGraphResolver

`independence_group`必须从来源图计算：

```text
ContentUnit
→ Document
→ Source
→ DerivationLink
→ Root Source
```

要求：

```text
同根Source → 同组
不同根Source → 不同组
环 → 失败
悬空 → 失败
跨Workspace → 失败
无法解析根Source → 失败
```

禁止：

```text
engine_<candidate_id>
Provider传入字符串
Document不同即自动独立
```

### B05：严格Mapper与引用解析

禁止填充：

```text
0
unknown
空ID
当前时间
默认fact_claim
默认EntityType
```

必要字段缺失：

```text
整笔事务失败
```

先建立：

```text
candidate_id → core_object_id
```

再映射Evidence等依赖对象。Evidence目标必须是实际核心对象ID，不得保留悬空Candidate ID。

### B06：持久化幂等自然键

至少将以下键写入`external_ids`或现有可查询字段：

```text
operation_key
draft_natural_key
origin_candidate_id
origin_review_bundle_sha256
```

禁止：

```text
仅扫描最近100条ProcessingRun
仅以candidate:<id>查重
仅以Entity名称查重
只在Python内存保存自然键
```

测试：

```text
关闭Session并重开
双Session先后执行
相同Bundle重放
等价新Bundle重放
超过100条ProcessingRun后重放
```

结果均不得产生严格重复对象。

若必须新增数据库唯一约束，先提出Migration子任务，不得隐式修改Alembic。

### M01：未知对象类型Fail-closed

未知`object_type`必须抛出明确错误。

禁止默认映射为Entity。

### M02：CI门禁改为硬失败

```text
冻结资产Manifest不存在
冻结资产Hash不匹配
未授权Schema变化
未授权Alembic变化
```

不得只打印WARN或skip。
