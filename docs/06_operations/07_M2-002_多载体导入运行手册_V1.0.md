# M2-002 多载体导入运行手册 V1.0

## 1. 安装

```bash
pip install -e ".[dev,parsers]"
```

## 2. 环境配置

```bash
export AURORA_DATABASE_URL="sqlite:///./data/aurora.db"
export AURORA_INGEST_MAX_BYTES=10485760
export AURORA_PDF_MAX_BYTES=52428800
export AURORA_PDF_MAX_PAGES=500
export AURORA_WEB_TIMEOUT_SECONDS=15
export AURORA_WEB_MAX_REDIRECTS=5
```

## 3. HTML

```bash
aurora-ingest html ./article.html \
  --source-name "Example Website" \
  --source-type official_website \
  --document-type web_article \
  --output json
```

指定主体：

```bash
aurora-ingest html ./article.html \
  --source-name "Example Website" \
  --source-type official_website \
  --content-selector "article .content"
```

## 4. URL

```bash
aurora-ingest url "https://example.com/article" \
  --source-name "Example Website" \
  --source-type official_website \
  --output json
```

默认禁止私网。只有明确的受控内网QA场景可使用：

```bash
--allow-private-network
```

生产环境不建议开启。

## 5. PDF

```bash
aurora-ingest pdf ./report.pdf \
  --source-name "Company Filing" \
  --source-type company_announcement \
  --document-type company_filing \
  --pages 15-18 \
  --table-mode best_effort \
  --output json
```

关闭表格：

```bash
--table-mode off
```

## 6. Transcript

```bash
aurora-ingest transcript ./captions.srt \
  --source-name "Video Channel" \
  --source-type video_channel \
  --document-type video \
  --output json
```

WebVTT同样使用`transcript`命令，格式默认由扩展名确定。

## 7. Dry-run

所有子命令均支持：

```bash
--dry-run
```

Dry-run执行采集、解析、Hash、身份和数据库读取，但不创建任何对象或ProcessingRun。

## 8. 重复策略

```text
reuse       默认，复用相同Document和ContentUnit
reject      重复时返回错误
new_version 强制新建版本
```

## 9. 状态

```text
parsed             完整解析
partially_parsed   有有效单元但存在质量警告
failed             无法建立有效文档
```

ProcessingRun对应：

```text
SUCCESS
PARTIAL_SUCCESS
FAILED
```

## 10. 常见错误

### WEB_PRIVATE_ADDRESS_BLOCKED

URL解析到私网、回环或链路本地地址。检查URL和DNS，不要通过关闭防护绕过未知地址。

### WEB_UNSUPPORTED_CONTENT_TYPE

URL返回的不是静态HTML，可能是PDF、JSON或下载流。

### PDF_NO_EXTRACTABLE_TEXT

PDF可能是扫描件。M2-002不包含OCR。

### PDF_INVALID_PAGE_RANGE

页码表达式不合法或超出文件物理页数。

### TRANSCRIPT_INVALID_TIMESTAMP

字幕时间格式错误或结束早于开始。

## 11. 日志安全

不得在日志中写入：

- Cookie；
- Authorization；
- 密码；
- Proxy凭据；
- 完整敏感正文。
