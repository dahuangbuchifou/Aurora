# M2-002 多载体解析实现说明 V1.0

## 1. 模块边界

```text
Collector：获取原始材料和传输元数据
Parser：确定性分段、Locator、Hash和质量报告
Workflow：身份、版本、事务、ProcessingRun和持久化
Repository：对象存取，不承担解析逻辑
```

## 2. Collector

### LocalBinaryFileCollector

用于PDF原始字节读取：

- 读取前检查大小；
- 空文件拒绝；
- 不做文本解码；
- 返回raw_bytes和file URI。

### StaticUrlCollector

职责：

- 校验HTTP(S) URL；
- 拒绝URL账号密码；
- DNS解析并检查所有IP；
- 每次redirect重新检查；
- 流式限制响应大小；
- 仅接受HTML Content-Type；
- 记录final URL、redirect chain、ETag和Last-Modified；
- 不保存Cookie或Authorization。

## 3. ParserConfig

```python
ParserConfig(
    name,
    version,
    options,
)
```

Hash算法：

```text
SHA-256(canonical JSON)
```

规范化JSON使用稳定键排序，不包含当前时间、随机值或临时文件路径。

## 4. HTML Parser

主体选择：

```text
content_selector → article → main → body
```

DOM清理：

```text
script/style/noscript/template/form/nav/footer/aside
```

结构提取按DOM顺序进行。父级结构节点中包含已支持子块时，不重复生成父段落文本。

Table处理：

```text
TABLE(parent)
└── TABLE_ROW(child)
```

`parent_sequence_no`在持久化前转成`parent_unit_id`。

## 5. PDF Parser

页码表达式：

```text
1
1,3,5
5-7
1,3,5-7
```

页码为1-based物理页。

表格：

- `off`：只处理页面文本；
- `best_effort`：提取表格并尽量从普通文本中排除重复表格行；
- 失败返回质量警告；
- 表格参数进入ParserConfigHash。

部分成功：

```text
存在有效单元 + 有质量警告
→ PARTIALLY_PARSED
```

## 6. Transcript Parser

支持：

```text
SRT
WebVTT
```

Cue字段：

```text
start_seconds
end_seconds
text
speaker
```

规则：

- end不得早于start；
- Cue start必须非递减；
- 重叠允许但产生警告；
- NOTE/STYLE/REGION块跳过；
- 空Cue丢弃并警告；
- 无有效Cue时失败。

## 7. Hash

### Raw Hash

```text
SHA-256(raw bytes)
```

### Semantic Hash

HTML：主要结构单元序列。

PDF：页码、单元类型、文本、表格行与父子关系。

Transcript：start_ms、end_ms、speaker和文本。

## 8. Workflow

```text
ProcessingRun PENDING
→ RUNNING
→ Collector
→ Parser
→ Source复用
→ Document身份计算
→ Document+ContentUnits原子提交
→ SUCCESS或PARTIAL_SUCCESS
```

Parser失败时不创建Document和ContentUnit，ProcessingRun记录FAILED。

## 9. 兼容

M2-001旧Document没有显式parser_config_hash时，`default`配置继续使用旧身份格式，避免旧对象被无意义地重新版本化。
