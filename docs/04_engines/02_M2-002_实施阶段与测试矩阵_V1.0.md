## 2. 单元测试

### Web

- 主体selector优先级；
- article/main/body fallback；
- script/nav/footer移除；
- title/canonical/published_at；
- malformed HTML；
- HTML table父子单元；
- CSS/XPath稳定；
- 空壳动态页；
- 相同主要内容、不同广告生成相同语义Hash。

### URL安全

- loopback；
- RFC1918私网；
- link-local；
- IPv6 loopback/private；
- DNS解析到私网；
- redirect到私网；
- 非HTTP协议；
- timeout；
- response size；
- Content-Type；
- 5次以上重定向。

### PDF

- 1-based page_no；
- 单页/多页；
- page range；
- 非法页码；
- 纯文本PDF；
- 表格PDF；
- 部分扫描页；
- 全扫描页；
- 加密PDF；
- 过大文件；
- 过多页；
- 读取顺序警告；
- table_mode影响config_hash。

### Transcript

- SRT；
- VTT；
- BOM；
- voice span；
- speaker前缀；
- 重叠Cue；
- 非递增时间；
- end < start；
- 空Cue；
- 多行Cue；
- 时间参与语义Hash。

## 4. E2E真实案例

### Case A Web

目标：

```text
中芯国际官网PR
→ URL或HTML快照
→ 标题/段落/表格
→ CSS/XPath Locator
→ SQLite
```

### Case B Transcript

将现有转写转换为 SRT/VTT Fixture：

```text
Cue
→ TRANSCRIPT_SEGMENT
→ start/end seconds
→ speaker
→ SQLite
```

### Case C PDF

使用年报选定页：

```text
PDF pages 15-18
→ Paragraph/Table/TableRow
→ page_no
→ SQLite
```

公开仓库只提交脱敏或合成Fixture；真实PDF可在QA环境使用。

## 5. 质量门禁

```text
Python 3.11.13
SQLite 3.26+
0 failed
0 warnings
总覆盖率 ≥ 90%
新增模块覆盖率 ≥ 90%
```

还必须：

- SHA256SUMS通过；
- Manifest通过；
- collect-only留档；
- 核心Schema目录diff为空；
- 不新增Alembic Revision，除非实现阶段发现无法避免并重新评审；
- 所有网络测试使用本地Server；
- 不在测试中访问真实外网。
