## 2. M1-003B 实现正文

### 2.1 核心 Schema V1.1

#### DataPoint

新增：

```python
measurement_context: MeasurementContext
```

支持：

```text
measurement_kind
currency
scale_multiplier
reporting_standard
attribution_scope
consolidation_scope
```

#### Claim

新增：

```python
claim_dimension: ClaimDimension = ClaimDimension.GENERAL
```

正式枚举包含：

```text
general
business_growth
financial_performance
valuation
risk
market_expectation
action_recommendation
policy
technology
operations
governance
supply_chain
competition
other
```

#### Provenance

新增：

```python
derivation_links: list[DerivationLink]
```

支持：

```text
derived_from
quoted_from
summarizes
reposts
calculated_from
translated_from
extracted_from
inferred_from
```

原 `origin_object_ids` 保留。

### 2.2 双版本兼容

新增：

```text
src/aurora/core/schema_migrations.py
```

行为：

- 读取 V1.0 时在内存中升级为 V1.1；
- 不自动写回数据库；
- 缺失的新字段使用保守默认值；
- 不猜测币种、报告标准或 ClaimDimension；
- V1.0 origin ID 映射为通用 `derived_from`；
- 未知版本明确失败。

### 2.3 JSON Schema

保留：

```text
schemas/v1/
```

新增：

```text
schemas/v1_1/
schemas/registry.json
schemas/examples/v1_1/
```

V1.1 Schema 对 `schema_version` 使用 `const: 1.1`。

### 2.4 Repository

新增：

```text
get_raw_record()
list_by_schema_version()
```

支持：

- 混合版本读取；
- 原始 payload 审计；
- 读取旧对象不写回；
- 显式更新旧对象后写入 V1.1。

### 2.5 Traceability

修复：

```text
dependency=false 且目标缺失
→ advisory reference
→ 不进入 dangling error
```

`processing_run_id` 缺失不再污染认知链完整性。

结构化 `derivation_links` 优先于旧 `origin_object_ids`。

### 2.6 Evidence 独立性

新增：

```text
group_independent_evidence()
count_independent_evidence()
validate_independence_groups()
```

独立证据键：

```text
target_object_id + evidence_role + independence_group
```

空组值在兼容逻辑中按 Evidence ID 视为独立组，不发生错误合并。

### 2.7 Claim 原子性

新增警告式：

```text
ClaimAtomicityLinter
```

检查：

- 多句主张；
- 预测与行动建议混合；
- 多个数值断言；
- 过长语句；
- 多从句标记。

Linter 不阻止保存。

### 2.8 EvidenceRole 规则

新增同范围直接性规则：

- 主体、时间、条件和维度一致时，才使用 `support/refute`；
- 维度不同优先 `qualify`；
- 主体或时间不同优先 `context`。

### 2.9 SQLite 与 Alembic

新增环境变量：

```text
AURORA_SQLITE_TIMEOUT_SECONDS=30
```

Engine 同时配置：

```text
connect_args.timeout
PRAGMA busy_timeout
```

新增 Alembic Revision：

```text
20260712_0002_add_schema_version_index
```

只增加 `schema_version` 索引，不修改 JSON payload。

### 2.10 Payload 迁移与恢复

新增命令：

```text
aurora-migrate-payloads
aurora-restore-payloads
```

支持：

- dry-run；
- JSONL 备份；
- SHA-256；
- 批次迁移；
- 单对象错误报告；
- 幂等重试；
- 备份恢复；
- 迁移报告。
