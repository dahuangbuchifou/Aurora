## 4. V1.1 核心变更

### DataPoint

新增可选嵌套对象：

```text
measurement_context
```

用于表达：

- 数据类型；
- 币种；
- 数量级；
- 报告或会计准则；
- 归属口径；
- 合并范围。

### Claim

新增：

```text
claim_dimension
```

`claim_type` 表示主张形式，`claim_dimension` 表示分析维度。

### Provenance

新增：

```text
derivation_links
```

结构化表达：

```text
quoted_from
summarizes
reposts
calculated_from
translated_from
extracted_from
inferred_from
derived_from
```

原有 `origin_object_ids` 保留，V1.1 不删除。

## 5. 兼容策略摘要

采用：

```text
加法式 Schema 升级
+ 双版本读取
+ 读取时适配
+ 混合版本持久化
+ 显式离线迁移
```

关键原则：

1. 不删除或重命名 V1.0 字段；
2. V1.0 对象可直接读取；
3. 读取旧对象时只在内存中升级，不自动改库；
4. 新建或显式更新的对象写入 V1.1；
5. 数据库可同时包含 V1.0 和 V1.1；
6. JSON payload 迁移不放入 Alembic；
7. Alembic 只处理 DDL，例如增加 `schema_version` 索引；
8. 离线迁移必须支持 dry-run、备份、校验和恢复。
