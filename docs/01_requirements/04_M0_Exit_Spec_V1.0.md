## 2. 第一阶段唯一主场景

> **用户（大黄）提交网页、PDF、文本或音视频资料，Aurora 自动完成内容解析，提取事实（Fact）、数据（DataPoint）、观点主张（Claim）、预测和证据（Evidence），形成结构化的知识卡片（KnowledgeObject）和正反观点矩阵，生成个人观点草案（PersonalOpinion Draft），并以 Markdown 格式输出研究简报。**

### 2.1 场景拆解

```text
用户获得一篇或多篇资料
  ↓
提交至 Aurora（文件 / URL / 笔记）
  ↓
Aurora 完成内容解析和文本提取
  ↓
AI 识别并提取
  - 事实性信息（Fact）
  - 数值数据（DataPoint）
  - 各主体的观点主张（Claim）
  - 预测与判断
  - 支撑证据（Evidence）
  ↓
Aurora 组织为知识卡片（KnowledgeObject）
  - 来源可追溯
  - 事实与观点分开放
  - 证据独立分组
  ↓
当同一主题有多条 Claim 时
  → 生成正反观点矩阵
  ↓
Aurora 生成个人观点草案（Draft）
  - 总结
  - 支持证据
  - 反方意见
  - 关键假设
  - 失效条件
  - 未知变量
  ↓
用户查看草案 → 确认、修正、或驳回
  ↓
确认后观点激活（active）
  ↓
输出 Markdown 研究简报
```

### 2.2 主场景边界

**M1 阶段只验证前半段**：从"提交资料"到"知识卡片+正反观点"的可结构化。

个人观点的确认激活和输出简报，在 M1 后期或 M2 再验证。

## 3. P0 输入与输出

### 3.1 输入（M1 优先级）

| 类型 | M1 | 说明 |
|------|:--:|------|
| Markdown / TXT | ✅ | 直接文本输入 |
| 网页 | ✅ | URL 抓取 + 正文提取 |
| PDF（文本型） | ✅ | 提取文本+页码 |
| 音视频（≤15min） | ✅ | Whisper 转写 |
| 音视频（>15min） | ❌ | M2 |
| PDF（扫描版） | ❌ | M3 |
| 社交媒体 | ❌ | M3 |

### 3.2 输出（M1 优先级）

| 输出 | M1 | 说明 |
|------|:--:|------|
| ContentUnit 列表 | ✅ | 含时间戳/页码定位 |
| Fact 列表 | ✅ | 经过证据确认的事实 |
| DataPoint 列表 | ✅ | 含口径和来源 |
| Claim 列表 | ✅ | 含说话人和上下文 |
| Evidence 列表 | ✅ | 含独立分组 |
| KnowledgeObject | ✅ | 结构化知识卡片 |
| 正反观点矩阵 | ✅ | 主题下多方观点对照 |
| PersonalOpinion Draft | ✅ | AI 生成草案，待确认 |
| Markdown 简报 | ✅ | 研究简报输出 |
| 可追溯原文 | ✅ | 每个结论可回原文位置 |

## 4. 非目标（明确不做）

以下能力在 MVP 及 M1 阶段明确不开发：

- ❌ 自动投资决策或交易建议；
- ❌ 多用户/多人协作/权限体系；
- ❌ 所有短视频平台自动采集；
- ❌ 复杂知识图谱/本体推理；
- ❌ AI 自动确认个人观点（`confirmed_by_user` 必须人工）；
- ❌ 实时监控/事件流/消息推送；
- ❌ 商业化部署/多租户；
- ❌ 移动端/Web UI（MVP 阶段命令行+Markdown 输出）；
- ❌ 超出 3 个测试案例的来源类型。

## 5. 核心术语

| 术语 | 定义 | 对应英文 |
|------|------|----------|
| 来源 | 信息的主体或渠道，如网站、作者、机构 | Source |
| 文档 | 具体的内容载体，如视频、文章、PDF | Document |
| 内容片段 | 可被定位和引用的最小内容单元 | ContentUnit |
| 实体 | 可独立识别的人、公司、行业、产品等 | Entity |
| 事件 | 特定时间发生的、影响实体的事件 | Event |
| 数据点 | 具有明确数值、口径、时间、来源的数据 | DataPoint |
| 主张 | 某主体提出的可被支持/反驳的观点 | Claim |
| 证据 | 支持、反驳或限定某结论的证据材料 | Evidence |
| 事实 | 经过证据支持、系统当前接受的事实性记录 | Fact |
| 知识对象 | 可长期复用的结构化知识单元 | KnowledgeObject |
| 洞察 | 基于多条知识形成的高阶推断 | Insight |
| 个人观点 | 用户确认的阶段性个人看法，含失效条件 | PersonalOpinion |
| 输出成果 | 为特定用途生成的研究简报等 | OutputArtifact |
| 反馈 | 对知识/观点/输出的修正反馈 | Feedback |

## 6. 三个测试案例

| # | 类型 | 主题 | 验证重点 |
|---|------|------|----------|
| A | 网页 | 半导体/芯片行业分析文章 | Fact/Claim/Evidence 分离、发布时间、作者归属 |
| B | 音视频 | 某行业分析视频（≤15min） | 说话人识别、时间戳定位、转写质量 |
| C | PDF | 公司年报/行业研报 | 页码定位、数据口径提取、事实与分析分离 |

**素材选择标准**：
- 可合法用于个人研究和测试；
- 发布时间、作者/发言人明确；
- 同时包含事实、数据和观点；
- 至少部分内容可交叉验证；
- 不含个人隐私（提交到仓库的必须脱敏）。

具体素材由项目负责人（大黄）挑选后，附在 M1-001 Issue 中。
