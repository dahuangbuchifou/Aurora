# M1-003 任务边界与启动条件 V1.0

> 建议名称：Aurora 核心对象模型 V1.1 评审、兼容升级与迁移  
> 当前状态：DRAFT  
> 启动前提：M1-002完成目标服务器QA。

## 1. 任务定位

M1-003不是继续扩功能，而是：

> 在进入M2自动化认知闭环前，修复已被真实案例证明的核心模型缺陷，并建立V1.0到V1.1的兼容机制。

## 2. 目标

1. 审查M1-002的BLOCKER和MAJOR问题；
2. 确定进入Schema V1.1的变更；
3. 保证V1.0数据可迁移或继续读取；
4. 完成Pydantic、JSON Schema、ORM和Migration升级；
5. 增加兼容和真实案例回归测试；
6. 冻结M2依赖的核心对象契约。

## 3. 启动条件

- [ ] M1-002目标服务器QA完成
- [ ] QA为PASS或可接受的CONDITIONAL PASS
- [ ] 三个案例无未解释悬空引用
- [ ] 审计报告和待优化清单已确认
- [ ] 婉儿给出MAJOR项产品优先级
- [ ] 大G给出兼容和Migration评估
- [ ] 项目负责人确认破坏性变更

## 4. 候选范围

### DataPoint财务口径

评估：

```text
currency
scale
accounting_standard
attribution_scope
consolidation_scope
value_type
```

重点决定通用Aurora模型与WQRS金融扩展的边界。

### Claim分析维度

评估：

```text
business_growth
valuation
risk
market_expectation
action_recommendation
policy
technology
```

不得与现有`claim_type`语义重叠。

### Provenance派生关系

评估：

```text
quoted_from
summarizes
reposts
calculated_from
translated_from
extracted_from
inferred_from
```

### Evidence独立性

实现或明确：

- 同一independence_group不重复计数
- 保留多个Document但聚合时去重
- 去重不等于删除来源

### Claim原子性

定义一条Claim只表达一个可支持或反驳的主判断，并补充测试。

### 工程补强

- schema_registry异常测试
- session和事务测试
- SQLite busy_timeout
- Schema兼容矩阵
- Migration升级/回退
- V1.0案例加载到V1.1的回归

## 5. 非目标

- 向量数据库、Web API、Web UI
- 自动投资建议、WQRS正式接入
