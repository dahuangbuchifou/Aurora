# M2-002：多载体解析适配器任务边界 V1.0

> **状态**：READY_FOR_DESIGN_REVIEW  
> **前置条件**：M2-001 CLOSED  
> **目标**：将网页、PDF和转写文件稳定映射为可定位的 ContentUnit。

---

## 1. 背景

M2-001 已建立：

```text
本地输入
→ Collector
→ Parser
→ Source
→ Document
→ ContentUnit
→ ProcessingRun
→ SQLite
```

但目前只支持 Markdown、纯文本和结构化分段 JSON。

M2-002 在不修改 Aurora V1.1 核心契约的前提下，增加三类真实研究载体：

```text
静态网页
机器生成PDF
带时间戳的转写文件
```

本任务只负责采集、解析、定位和质量报告，不进入事实与观点理解。

---

## 2. 目标

实现：

1. 本地 HTML 文件解析；
2. 显式单 URL 静态网页抓取与解析；
3. 本地机器生成 PDF 文本解析；
4. PDF best-effort 表格提取；
5. SRT 和 WebVTT 转写解析；
6. 页面、DOM、时间戳级来源定位；
7. ParseReport 和质量警告；
8. 幂等、版本、事务和 ProcessingRun；
9. CLI text/json 输出；
10. M1-002 三个案例的真实载体回归。

---

## 3. 输入范围

### 3.1 Web

支持：

```text
.html
.htm
http://
https://
```

URL 仅支持用户显式提交的单个地址。

不支持：

- 站点爬取；
- 链接递归；
- 登录；
- Cookie导入；
- JavaScript渲染；
- CAPTCHA；
- 反爬绕过；
- 浏览器自动化；
- 需要交互的页面。

### 3.2 PDF

支持：

```text
.pdf
```

限定：

- 机器生成或具有可提取文本层；
- 本地文件；
- 可指定页码范围；
- 可输出文本、表格和质量警告。

不支持：

- OCR；
- 图片型扫描件识别；
- 手写内容；
- PDF编辑；
- 受损文件修复；
- 复杂密码管理。

加密 PDF 默认拒绝。后续如需支持，必须使用安全的密码输入机制，不能通过普通日志记录。

### 3.3 Transcript

支持：

```text
.srt
.vtt
```

现有 Structured Segments JSON 继续有效。

一条字幕 Cue 默认映射为一个：

```text
ContentUnitType.TRANSCRIPT_SEGMENT
```

不在 M2-002 合并成语义段落。上下文窗口和语义合并属于 M2-003。

## 4. Web解析规则

### 4.1 主体区域选择

优先级：

```text
显式CSS Selector
→ <article>
→ <main>
→ 语义内容候选
→ <body>
```

默认移除：

```text
script
style
noscript
template
form
nav
footer
aside
```

`header` 不全局删除，防止误删文章标题；由主体选择规则决定。

### 4.2 元数据

尝试提取：

```text
<title>
<link rel="canonical">
og:title
og:site_name
article:published_time
author
lang
```

所有推断值必须带来源说明，不得覆盖用户显式输入。

### 4.3 HTML单元

支持：

```text
HEADING
PARAGRAPH
LIST_ITEM
QUOTE
CODE_BLOCK
TABLE
TABLE_ROW
```

HTML Table：

- 建立一个 TABLE 父单元；
- 每行建立 TABLE_ROW 子单元；
- 单元顺序按 DOM 出现顺序；
- 表头、caption 和单元格按确定性文本格式保存；
- 不使用 DataFrame 作为核心持久化格式。

### 4.4 Locator

每个 HTML 单元至少包含：

```text
css_selector 或 xpath
block_no
heading_path（适用时）
```

CSS Selector 和 XPath 必须针对保存的 HTML 快照可重放。

## 5. PDF解析规则

### 5.1 页码

Aurora `SourceLocator.page_no` 使用：

```text
PDF文件中的1-based物理页序号
```

不使用印刷页码作为主定位字段。

若页面中存在印刷页码，可作为应用元数据保存，但不替代 `page_no`。

### 5.2 文本单元

基础单位：

```text
HEADING（高置信度时）
PARAGRAPH
LIST_ITEM（可稳定识别时）
TABLE
TABLE_ROW
```

默认不根据字体大小强制认定所有标题。低置信度布局保持为 PARAGRAPH 并产生警告。

### 5.3 表格

M2-002要求：

```text
对机器生成PDF进行best-effort表格提取
```

规则：

- 固定表格提取参数并纳入 parser_config_hash；
- 每个表生成 TABLE 父单元；
- 每行生成 TABLE_ROW；
- 表格文本从页面普通段落中尽量排除，降低重复；
- 不能可靠提取时保留页面文本并产生警告；
- 不宣称 PDF 表格100%准确；
- 表格提取失败不自动导致整份文档 FAILED。

### 5.4 部分解析

以下情况可判定：

```text
PARTIALLY_PARSED
```

- 部分页面无文本；
- 表格提取失败但正文成功；
- 部分页面出现顺序不确定；
- 只解析指定页码范围；
- 存在扫描页，但其他页可提取。

若全文件没有有效 ContentUnit，则：

```text
FAILED
```

### 5.5 页码范围

CLI允许：

```text
--pages 15-18
--pages 1,3,5-7
```

页码选择必须进入：

```text
parser_config_hash
```

否则同一 PDF 的不同页码选择可能被错误复用为同一 Document。

## 6. Transcript解析规则

### 6.1 Cue映射

每个 Cue 生成：

```text
unit_type = transcript_segment
start_seconds
end_seconds
block_no
speaker（可识别时）
```

### 6.2 Speaker

优先级：

```text
WebVTT voice span
→ 显式speaker字段
→ “Speaker:”前缀
→ null
```

Parser只能提取明确存在的说话人，不进行说话人推断。

### 6.3 时间规则

- 起止时间必须可解析；
- `end_seconds >= start_seconds`；
- Cue 顺序应非递减；
- 重叠 Cue 可以保留，但必须产生质量警告；
- 空 Cue 丢弃并记录警告；
- 时间戳不得因文本清洗而改变。

## 7. 幂等性升级

M2-001 文档身份：

```text
source_id
+ semantic_content_hash
+ parser_name
+ parser_version
```

M2-002扩展为：

```text
source_id
+ semantic_content_hash
+ parser_name
+ parser_version
+ parser_config_hash
```

`parser_config_hash` 至少覆盖：

```text
页面范围
HTML主体selector
PDF table_mode
文本布局参数
Transcript格式
明确影响分段结果的其他选项
```

兼容规则：

```text
M2-001历史对象缺少parser_config_hash
→ 视为默认配置hash
→ 不修改V1.1核心Schema
```

## 8. 内容Hash

每次采集生成：

```text
raw_content_hash
semantic_content_hash
```

### raw_content_hash

基于原始响应或文件字节，用于采集审计。

### semantic_content_hash

基于确定性规范化后的主要文本、表格或 Cue 序列，用于 Document 幂等。

规则：

- HTML广告或脚本变化不应轻易制造新文档；
- PDF二进制元数据变化但提取内容相同，可复用；
- Transcript时间戳变化应改变语义Hash；
- Parser版本或配置变化仍会形成新Document版本。

---

## 9. ProcessingRun

每次解析必须记录：

```text
collector
parser_name
parser_version
parser_config_hash
raw_content_hash
semantic_content_hash
input_bytes
unit_count
warning_count
duration_ms
```

失败时：

- Document和ContentUnit回滚；
- ProcessingRun保留 FAILED；
- 错误码稳定；
- 日志不得包含Cookie、密码或完整敏感正文。

## 10. CLI

建议新增：

```bash
aurora-ingest html ./article.html
aurora-ingest url https://example.com/article
aurora-ingest pdf ./report.pdf --pages 15-18
aurora-ingest transcript ./captions.vtt
```

PDF表格模式：

```bash
--table-mode off
--table-mode best_effort
```

默认：

```text
best_effort
```

Web主体覆盖：

```bash
--content-selector "article .content"
```

输出继续支持：

```text
--output text
--output json
```
