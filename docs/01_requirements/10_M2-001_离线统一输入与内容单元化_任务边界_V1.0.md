# M2-001：离线统一输入与内容单元化任务边界 V1.0

> 状态：READY_FOR_SPEC_REVIEW  
> 阶段：M2  
> 目标：建立不依赖网络和LLM的确定性输入基础。

## 1. 背景

M1已冻结V1.1契约，但当前对象主要由Fixture和人工JSON创建。

M2-001建立第一条真实输入链：

```text
本地文件或结构化文本
→ 读取
→ 校验
→ 确定性分段
→ Source / Document / ContentUnit
→ ProcessingRun
→ SQLite
```

## 2. 目标

支持：

1. 本地Markdown；
2. 本地纯文本；
3. 结构化分段JSON；
4. 确定性ContentUnit；
5. 内容hash；
6. 幂等导入；
7. ProcessingRun；
8. SQLite持久化；
9. CLI；
10. 错误报告和重试。

## 3. 输入与模块范围

### 应用DTO

建议新增但不加入核心对象注册表：

```text
IngestionRequest
LocalFileInput
StructuredSegmentsInput
IngestionResult
IngestionError
```

主要字段：

```text
workspace_id
source_name
source_type
input_type
path
document_type
title
published_at
language
tags
idempotency_key
parser_options
```

### Collector

支持：

```text
local_file
inline_text
structured_segments
```

Collector只读取内容和基础元数据，不做语义提取。

### Parser

支持：

```text
text/plain
text/markdown
application/vnd.aurora.segments+json
```

Markdown最少识别：

- heading；
- paragraph；
- list item；
- quote；
- code block。

纯文本按空行分段。

Structured Segments JSON支持：

- unit_type；
- text；
- sequence_no；
- locator；
- speaker。

### Workflow

```text
ProcessingRun(PENDING)
→ RUNNING
→ Collector
→ 计算hash
→ 幂等检查
→ 创建/复用Source
→ 创建/复用Document
→ Parser生成ContentUnit
→ 原子持久化
→ ProcessingRun SUCCESS
```

失败时：

```text
ProcessingRun FAILED
+ error_code
+ error_message
```

### CLI

```bash
aurora-ingest file ./example.md   --source-name "Local Research"   --source-type local_file   --document-type markdown   --workspace default
```

```bash
aurora-ingest segments ./segments.json
aurora-ingest file ./example.md --dry-run
```

## 4. 幂等性

内容hash：

```text
SHA-256(normalized_bytes)
```

Idempotency Key优先级：

```text
显式idempotency_key
→ normalized source URI + content_hash
→ absolute file URI + content_hash
```

重复策略支持：

```text
reuse
reject
new_version
```

默认：

```text
reuse
```

重复执行时：

- 不重复创建Document和ContentUnit；
- 创建新的ProcessingRun；
- 指向已存在对象；
- 返回`reused=true`。

## 5. ContentUnit稳定性

要求：

- sequence_no规范统一；
- 同一输入、同一Parser版本产生相同分段；
- locator可回到原文件；
- parser_version进入ProcessingRun.processor；
- ContentUnit ID必须确定；
- Parser版本变化时产生新Document版本，不静默覆盖。

建议ID输入：

```text
document_id
unit_type
sequence_no
normalized_text_hash
```

## 6. 事务和失败语义

单次任务内：

```text
Document + ContentUnits
```

必须原子提交。

失败时：

- 不留下半套ContentUnit；
- ProcessingRun记录FAILED；
- Source可以复用；
- 默认不启用partial模式；
- 重试生成新的ProcessingRun。
