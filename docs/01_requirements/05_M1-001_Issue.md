## 背景

Aurora 的核心设计建立在一套严格的分层对象模型之上。在开发任何采集器、解析器或 AI 理解模块之前，必须先把"数据宪法"落地为可运行的代码，确保：

1. 对象边界清晰且可校验；
2. JSON Schema 可对外发布；
3. SQLite 可正确持久化和读取；
4. 对象之间的关联关系（来源追溯、证据链、版本）可验证。

---

## 用户场景

> 作为 Aurora 系统，我需要将"来源 → 文档 → 内容片段 → 事实/数据/观点/证据 → 知识对象 → 洞察 → 个人观点 → 输出成果 → 反馈"这 15 个核心对象定义为标准的 Pydantic 模型，并能够保存到 SQLite 中，以支撑后续所有认知处理引擎的开发。

## 业务规则

### 7.1 所有 15 个核心对象业务规格

#### Source（来源）
```python
字段: id, name, source_type, publisher, author, homepage_url, domain,
      source_quality_tier(S1-S5), ownership_or_interest, status(active/inactive/archived),
      privacy_level, provenance, 通用字段全部
```

#### Document（文档）
```python
字段: id, source_id(fk→Source), document_type, title, original_url,
      published_at, collected_at, authors[], content_hash, mime_type,
      raw_storage_uri, parse_status, copyright_note, privacy_level,
      status(active/archived/superseded等), provenance, 通用字段全部
```

#### ContentUnit（内容片段）
```python
字段: id, document_id(fk→Document), unit_type, sequence_no, text,
      locator(JSON: start_seconds/end_seconds 或 page_no/block_no),
      speaker, language, quality(parse_quality, asr_confidence),
      status, provenance, 通用字段全部
```

#### Entity（实体）
```python
字段: id, entity_type, canonical_name, aliases[], external_ids(JSON),
      attributes(JSON), status, provenance, 通用字段全部
```

#### Event（事件）
```python
字段: id, event_type, title, event_time(start/end/precision),
      entities[](fk→Entity), location, description, source_refs[],
      status(reported/confirmed/disputed等), provenance, 通用字段全部
```

#### DataPoint（数据点）
```python
字段: id, metric, value(float), unit, entity_id(fk→Entity),
      period(start/end), reported_at, calculation_method,
      source_ref(fk→ContentUnit), status, provenance, 通用字段全部
```

#### Claim（主张）
```python
字段: id, claim_type, statement, subject_entity_ids[](fk→Entity),
      asserted_by(fk→Entity), asserted_at, time_horizon(start/end),
      conditions[], source_ref(fk→ContentUnit),
      epistemic_status(asserted/supported/disputed等),
      status, provenance, 通用字段全部
```

#### Evidence（证据）
```python
字段: id, evidence_role(support/refute/qualify/context/counterexample),
      evidence_type, target_object_id(多态), source_ref(fk→ContentUnit),
      summary, independence_group, directness, source_quality_tier,
      status, provenance, 通用字段全部
```

#### Fact（事实）
```python
字段: id, statement, subject_entity_ids[](fk→Entity),
      valid_time(start/end), evidence_ids[](fk→Evidence),
      verification_status, reviewed_by, status, provenance, 通用字段全部
```

#### KnowledgeObject（知识对象）
```python
字段: id, knowledge_type, title, topic_ids[](fk→Entity), summary,
      fact_ids[](fk→Fact), data_point_ids[](fk→DataPoint),
      claim_ids[](fk→Claim), evidence_ids[](fk→Evidence),
      related_entity_ids[](fk→Entity), validity(as_of_date/review_due_at),
      knowledge_status, status, provenance, 通用字段全部
```

#### Relation（关系）
```python
字段: id, subject_id(多态), predicate, object_id(多态),
      valid_time(start/end), evidence_ids[](fk→Evidence),
      status, provenance, 通用字段全部
```

#### TimelineEntry（时间线条目）
```python
字段: id, target_object_id(多态), event_date, event_type, title,
      summary, source_ref, status, provenance, 通用字段全部
```

#### Insight（洞察）
```python
字段: id, title, statement, supporting_object_ids[](多态),
      counter_evidence_ids[](fk→Evidence), reasoning_steps[],
      alternative_explanations[], confidence_level(low/medium/high),
      validity(as_of_date/review_due_at),
      insight_status(draft→active→archived等), status, provenance, 通用字段全部
```

#### PersonalOpinion（个人观点）
```python
字段: id, title, statement, topic_ids[](fk→Entity),
      supporting_ids[](多态), counter_evidence_ids[](fk→Evidence),
      key_assumptions[], unknown_variables[], invalidation_conditions[],
      confidence_level, opinion_status(draft→active→revised等),
      confirmed_by_user(bool), confirmed_at, review_due_at,
      version_no, previous_version_id, status, provenance, 通用字段全部
```

#### OutputArtifact（输出成果）
```python
字段: id, output_type, title, purpose, audience, content_uri,
      knowledge_refs[](fk→KnowledgeObject), insight_refs[](fk→Insight),
      opinion_refs[](fk→PersonalOpinion), as_of_date,
      review_status, status, provenance, 通用字段全部
```

#### Feedback（反馈）
```python
字段: id, feedback_type, target_object_id(多态), content,
      effect, source_refs[], submitted_by, status, provenance, 通用字段全部
```

#### ProcessingRun（处理记录）
```python
字段: id, run_type, started_at, completed_at, status(success/failed等),
      input_count, artifact_count, error_log, provenance, 通用字段全部
```

### 7.2 通用字段（每个对象必需）

```python
id: str = Field(pattern=r'^(src|doc|cu|ent|evt|dat|clm|evi|fac|knw|rel|tle|ins|opn|out|fbk|run)_[a-f0-9-]+')
object_type: str          # 固定值，如 "source"
schema_version: str       # 固定 "1.0"
status: str               # active/archived/deleted
created_at: datetime      # UTC
updated_at: datetime      # UTC
created_by: str           # "user" / "system" / module name
tags: list[str]           # 可选
source_refs: list[str]    # 可选，关联来源对象ID
provenance: dict          # 来源和处理链
privacy_level: str        # public/private/sensitive
deleted_at: datetime|None # 软删除
```

### 7.3 关键业务约束

1. **ID 格式严格遵守前缀规范**：`src_`、`doc_`、`cu_`、`ent_`、`evt_`、`dat_`、`clm_`、`evi_`、`fac_`、`knw_`、`rel_`、`tle_`、`ins_`、`opn_`、`out_`、`fbk_`、`run_`
2. **Claim ≠ Fact**：Claim 保留 `asserted_by`，Fact 需要 `verification_status` 且不低于 `verified`
3. **Evidence 必须有 `independence_group`**：同源内容归入同一组
4. **PersonalOpinion 激活条件**：`confirmed_by_user=true` 且 `key_assumptions` 非空且 `invalidation_conditions` 非空
5. **DataPoint 必须有 unit/time/口径**：无这些字段的数值不可作为 DataPoint
6. **ContentUnit 必须有 locator**：无定位信息的派生对象不得标记高证据质量
7. **时间统一 UTC ISO 8601**
8. **金额必须含 currency + unit**

## 异常场景

| 场景 | 预期行为 |
|------|----------|
| ID 格式不符合前缀规范 | Pydantic ValidationError |
| 缺少通用字段 | Pydantic ValidationError |
| status 使用非法值 | Pydantic ValidationError |
| DataPoint 缺少 unit | Pydantic ValidationError |
| PersonalOpinion confirmed_by_user=true 但 assumptions 为空 | Pydantic ValidationError |
| 外键引用不存在的 ID | SQLite ForeignKeyConstraintError（集成测试验证） |
| 重复 ID | IntegrityError |
| ContentUnit locator 缺失 | 允许创建但 `quality.parse_quality` 应为 low |
| 来源质量 tier 非法值 | Pydantic ValidationError |
