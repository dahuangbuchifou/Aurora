# M2-002：解析架构与来源定位契约 V1.0

---

## 1. 总体架构

```text
Input Request
    ↓
Collector Registry
    ├── LocalHtmlCollector
    ├── StaticUrlCollector
    ├── LocalPdfCollector
    └── LocalTranscriptCollector
    ↓
CollectedArtifact
    ↓
Parser Registry
    ├── HtmlDocumentParser
    ├── PdfDocumentParser
    ├── SrtTranscriptParser
    └── WebVttTranscriptParser
    ↓
ParsedDocument + ParseReport
    ↓
M2-001 IngestionWorkflow
    ↓
Source / Document / ContentUnit / ProcessingRun
```

Collector负责获得原始内容。

Parser负责：

- 确定性解析；
- 分段；
- Locator；
- 质量警告；
- 语义Hash。

Parser不得：

- 创建Fact或Claim；
- 调用LLM；
- 判断内容真假；
- 自动改变Source质量等级。

---

## 2. 注册机制

建议新增：

```text
CollectorRegistry
ParserRegistry
```

路由键：

```text
transport
media_type
file_extension
explicit_input_type
```

优先级：

```text
用户显式input_type
→ 响应Content-Type
→ 文件魔数
→ 文件扩展名
```

扩展名与魔数冲突时拒绝，不静默猜测。

---

## 3. CollectedArtifact

建议应用DTO：

```python
CollectedArtifact:
    transport
    source_uri
    media_type
    raw_bytes
    decoded_text
    raw_content_hash
    byte_length
    response_metadata
```

`response_metadata`可包含：

```text
HTTP status
final URL
Content-Type
ETag
Last-Modified
redirect chain
PDF file metadata
```

不得保存：

- Cookie；
- Authorization；
- Proxy密码；
- 完整请求头中的敏感字段。

## 4. ParsedDocument升级

M2-001现有 ParsedDocument 保持兼容，新增可选字段：

```python
ParsedDocument:
    parser
    content_hash
    units
    inferred_title
    structured_manifest
    parse_report
    document_metadata
```

其中：

```text
content_hash = semantic_content_hash
```

### ParseReport

```python
ParseReport:
    status
    raw_content_hash
    semantic_content_hash
    parser_config_hash
    unit_count
    warning_codes
    metrics
```

`status`建议：

```text
parsed
partially_parsed
failed
```

## 5. ParsedUnit升级

新增应用层字段：

```python
ParsedUnit:
    sequence_no
    unit_type
    text
    locator
    speaker
    parent_sequence_no
    quality_flags
```

`parent_sequence_no`用于：

```text
TABLE
  └── TABLE_ROW
```

Workflow持久化前：

1. 先计算全部ContentUnit ID；
2. 建立 sequence_no → ContentUnit ID 映射；
3. 将 `parent_sequence_no` 转为 `parent_unit_id`；
4. 一个事务写入。

该字段不进入核心对象注册表。

## 6. Locator映射

### HTML

| 内容 | SourceLocator |
|---|---|
| Heading | css_selector/xpath + block_no + heading_path |
| Paragraph | css_selector/xpath + paragraph_no + heading_path |
| List item | css_selector/xpath + block_no |
| Quote | css_selector/xpath + block_no |
| Table | css_selector/xpath + block_no |
| Table row | css_selector/xpath + block_no + row_no |

### PDF

| 内容 | SourceLocator |
|---|---|
| Page paragraph | page_no + paragraph_no + block_no |
| Heading | page_no + block_no + heading_path |
| Table | page_no + block_no |
| Table row | page_no + block_no + row_no |

### Transcript

| 内容 | SourceLocator |
|---|---|
| Caption cue | start_seconds + end_seconds + block_no |

## 8. PDF精确坐标决策

V1.1 SourceLocator没有：

```text
x0 / y0 / x1 / y1
```

M2-002不修改核心Schema。

暂定：

- 使用 page_no + block_no + row_no；
- Parser内部可保留bbox用于去重；
- 不将bbox写入任意自由 metadata 作为长期契约；
- 真实案例证明页级定位不足时，建立独立Schema Issue。

## 9. 版本链

以下变化创建新Document版本：

```text
parser_version变化
parser_config_hash变化
semantic_content_hash变化
```

父版本选择：

```text
相同Source
+ 相同原始URI或文件身份
+ 最近的已成功Document
```

禁止：

- 覆盖旧ContentUnit；
- 修改旧Locator；
- 读取时自动重解析。

---

## 10. 事务

单次导入：

```text
Source：可独立复用
ProcessingRun：独立状态事务
Document + ContentUnits：业务原子事务
```

Parser失败：

```text
ProcessingRun = FAILED
Document = 不创建
ContentUnit = 不创建
```

部分解析成功：

```text
Document.parse_status = PARTIALLY_PARSED
有效ContentUnit = 提交
ParseReport = 记录警告
ProcessingRun = SUCCESS（含warning）
```

只有存在至少一个有效单元时，才允许部分解析提交。
