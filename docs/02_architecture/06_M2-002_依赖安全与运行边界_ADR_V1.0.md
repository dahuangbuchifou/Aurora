# M2-002：依赖、安全与运行边界 ADR V1.0

---

## ADR-018：HTML使用Beautiful Soup与lxml

### 状态

```text
PROPOSED
```

### 决定

采用：

```text
beautifulsoup4
lxml
```

理由：

- 能处理真实世界中不完全规范的HTML；
- DOM遍历和CSS查询清晰；
- 便于生成可重放Locator；
- 不需要浏览器运行时。

不采用：

```text
Playwright / Selenium
```

原因：

- M2-002不执行JavaScript；
- 浏览器依赖和资源成本过高；
- 会扩大登录、反爬和动态交互边界。

## ADR-019：显式单URL使用HTTPX同步客户端

### 状态

```text
PROPOSED
```

### 决定

采用同步 HTTP 客户端：

```text
httpx
```

M2-002为单任务CLI，不需要异步爬虫。

冻结限制：

```text
timeout = 15s
max_redirects = 5
max_html_bytes = 10 MiB
max_pdf_bytes = 50 MiB
User-Agent = Aurora/<version>
```

不启用：

- 自动Cookie持久化；
- 环境代理自动继承；
- .netrc；
- 登录认证；
- HTTP/2强制；
- 并发抓取。

## ADR-020：静态URL默认阻止私有网络

### 状态

```text
PROPOSED
```

### 决定

默认拒绝：

```text
localhost
loopback
private network
link-local
multicast
unspecified
file://
ftp://
非HTTP(S)
```

DNS解析前后和每次重定向都必须重新校验。

理由：

- 防止SSRF；
- 防止URL导入器访问云元数据地址；
- 防止公开内容参数意外读取内网。

后续如确需内网页面，必须通过显式配置启用，且配置默认关闭。

## ADR-021：PDF使用pdfplumber

### 状态

```text
PROPOSED
```

### 决定

采用：

```text
pdfplumber
```

范围：

- 文本；
- 页面布局；
- 机器生成PDF表格；
- 页级定位；
- 调试Fixture。

原因：

- 同一依赖可提供文本、布局和表格能力；
- 适合机器生成PDF；
- 不需要外部Java服务；
- Python 3.11可运行。

明确限制：

- 不包含OCR；
- 扫描件效果有限；
- 不保证所有PDF表格正确；
- 表格参数必须版本化和测试。

## ADR-022：字幕使用webvtt-py

### 状态

```text
PROPOSED
```

### 决定

采用：

```text
webvtt-py
```

支持：

```text
WebVTT
SubRip/SRT
```

Aurora负责：

- 时间字符串转秒；
- Cue顺序校验；
- speaker清洗；
- 错误码；
- ContentUnit映射。

不使用字幕库的自动分段作为默认行为，保持一Cue一单元。

## ADR-023：解析依赖使用可选Extra

### 状态

```text
PROPOSED
```

### 决定

建议：

```toml
[project.optional-dependencies]
parsers = [
  "beautifulsoup4",
  "lxml",
  "httpx",
  "pdfplumber",
  "webvtt-py"
]
```

安装：

```bash
pip install -e ".[dev,parsers]"
```

理由：

- 基础对象层不强依赖解析器；
- M1测试仍可在最小依赖环境运行；
- 不同部署可按能力安装。

## ADR-024：PDF页数和资源限制

### 状态

```text
PROPOSED
```

### 决定

默认：

```text
PDF文件：50 MiB
PDF页数：500页
HTML响应：10 MiB
Transcript：10 MiB
```

均允许配置覆盖。

处理流程：

1. 先检查文件或响应大小；
2. 打开PDF后检查页数；
3. 超限前不做完整解析；
4. 记录FAILED ProcessingRun；
5. 不留下Document和ContentUnit。

## ADR-025：不修改V1.1 SourceLocator

### 状态

```text
PROPOSED
```

### 决定

M2-002使用现有字段：

```text
page_no
block_no
paragraph_no
row_no
heading_path
css_selector
xpath
start_seconds
end_seconds
```

不增加PDF bbox核心字段。

理由：

- 页级定位对MVP证据回溯已足够；
- 当前没有真实验收证明bbox是BLOCKER；
- V1.1已经冻结；
- 避免为单一Parser改变核心契约。
