## 5. 关键实现决策

### 5.1 HTML

主体选择优先级：

```text
显式content_selector
→ article
→ main
→ body
```

支持：

```text
HEADING
PARAGRAPH
LIST_ITEM
QUOTE
CODE_BLOCK
TABLE
TABLE_ROW
```

每个单元保存 CSS Selector、XPath、block_no 和 heading_path，能够对保存的 HTML 快照执行重放。

### 5.2 Static URL

只采集用户显式提交的单个 HTTP(S) 地址。

安全措施：

- URL协议白名单；
- 禁止URL内嵌账号密码；
- DNS解析结果检查；
- 每次重定向重新检查；
- 默认阻止私网、回环、链路本地、多播和保留地址；
- `trust_env=False`，不继承环境代理或 `.netrc`；
- timeout、重定向和流式响应大小限制；
- 只接受静态 HTML Content-Type。

### 5.3 PDF

- 仅解析具有文本层的机器生成 PDF；
- `page_no` 为 PDF 文件内 1-based 物理页；
- 支持 `1,3,5-7` 页码表达式；
- 表格采用 best-effort；
- 表格生成 TABLE 父单元和 TABLE_ROW 子单元；
- 表格失败可保留页面正文并返回质量警告；
- 全文无有效单元时明确失败；
- 不做 OCR，也不修改 SourceLocator 增加 bbox。

### 5.4 Transcript

- 支持 SRT 和 WebVTT；
- 一 Cue 一 `TRANSCRIPT_SEGMENT`；
- 保留 start/end seconds；
- 提取明确的 WebVTT voice 或 `Speaker:` 前缀；
- 重叠 Cue 保留并标记警告；
- 非递增 Cue 和非法时间戳明确失败；
- 时间戳参与 Semantic Hash。

### 5.5 身份与版本

Document 身份扩展为：

```text
source_id
+ semantic_content_hash
+ parser_name
+ parser_version
+ parser_config_hash
```

配置 Hash 覆盖：

```text
HTML content_selector
PDF page_selection
table_mode
max_pages
Transcript format和cue_granularity
```

配置、内容或 Parser 版本变化都会建立新 Document 版本，不覆盖旧数据。

## 7. 已知限制

- 动态 JavaScript 页面可能只返回空壳 HTML；
- PDF 表格质量依赖原文布局；
- 扫描 PDF 没有 OCR；
- PDF Locator 目前为页/块/行级，不含像素坐标；
- 字幕不做语义合并；
- URL抓取不支持登录、Cookie和反爬绕过；
- 未实现跨进程并发导入锁。
